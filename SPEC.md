# 感情ラベリング練習アプリ 仕様書(ドラフト v0.1)

## 1. 目的

「感情ラベリング」——相手の話から相手の感情を言葉にして返す傾聴テクニック——を、
一人でスマホで練習できるようにする。

鍛えたいのは次の 2 つ:

1. **感情語彙の引き出し**: 「大変でしたね」のような万能ワードに逃げず、状況に合った感情語を複数思いつける
2. **選択の精度**: 複数の候補の中から、相手に最も「刺さる」ラベルを選び分けられる

用途は個人の練習用。アカウント・サーバー不要で、オフラインで完結することを目指す。

## 2. コア練習ループ

```
┌─ 1. 出題 ──────────────────────────────┐
│ 状況説明(誰が・どんな場面で)+ 相手の発話    │
└──────────────────────────────────────┘
              ↓
┌─ 2. 回答(自由記述) ────────────────────┐
│ 相手に返すラベリングの一言を自分の言葉で入力  │
│ 例:「それは悔しかったですね」              │
│ ※感情語単体ではなく「実際に返す一言」で決定  │
└──────────────────────────────────────┘
              ↓
┌─ 3. 答え合わせ ─────────────────────────┐
│ 用意された感情語の候補セットを開示:         │
│  ◎ ベスト   … 最も刺さるラベル + 理由      │
│  ○ 許容    … 間違いではないが精度が落ちる   │
│  △ ズレ    … ありがちな外し方 + なぜ外れるか │
│ + 発話中の「手がかり」の解説               │
│ + 模範応答例                             │
└──────────────────────────────────────┘
              ↓
┌─ 4. 採点・記録 ─────────────────────────┐
│ 回答に含まれる感情語を候補と照合して自動判定。 │
│ 照合できない場合は自己採点(◎/○/×)。       │
│ 結果を記録し、外した問題は復習キューへ。      │
└──────────────────────────────────────┘
```

## 3. シナリオデータ

事前作成のシナリオ集をアプリに同梱する(JSON)。

```json
{
  "id": "s001",
  "category": "職場",
  "difficulty": 1,
  "context": "同僚が退勤間際に、ため息まじりに話しかけてきた。",
  "utterance": "今日さ、部長にみんなの前で企画書のこと詰められてさ…。俺だけじゃなくて、チーム全員で作ったやつなのに。",
  "labels": {
    "best": [
      { "word": "悔しい", "synonyms": ["悔しさ", "不本意"], "why": "『俺だけじゃなくて』に、不当に一人で責められた無念さがにじんでいる" }
    ],
    "ok": [
      { "word": "腹立たしい", "synonyms": ["ムカつく", "憤り"], "why": "怒りも含まれるが、核心は不公平への無念さなので一歩浅い" }
    ],
    "poor": [
      { "word": "悲しい", "synonyms": [], "why": "落ち込み一般に流してしまい、『不当さ』の感覚を拾えていない" }
    ]
  },
  "cues": [
    "「みんなの前で」→ 屈辱・面子",
    "「俺だけじゃなくて」→ 不公平感・無念さ"
  ],
  "modelReply": "それは悔しいですね。チームみんなで作ったのに、一人だけ言われたら。"
}
```

### シナリオの供給源は 2 系統

| | A: 会話シナリオ(創作) | B: 小説モード(青空文庫) |
|---|---|---|
| 出題形式 | 相手の発話に返す一言を考える | 場面を読み、登場人物の感情を言い当てる |
| 鍛える力 | ラベリングを「言う」力 | 感情を「読み取る」力 |
| 正解の根拠 | 監修者の判断(主観が入る) | 作者が本文中に明記した感情語(客観) |
| 作り方 | LLM 下書き+人手監修 | 機械抽出+人手フィルタ(§3.1) |

### 3.1 小説モード(B)の作成パイプライン

小説から、感情が明記されているシーンを抜き出し、その感情語を正解とする。

テキストの供給源(検討の結果):

- **青空文庫**: 主力。著作権切れの最前線(〜1967年没)には文体が現代的な作家がいる。
  優先候補: 山川方夫(現代的な短編)、太宰治(口語の一人称独白)、
  山本周五郎(感情描写の質は最高級だが時代物)。
  ※1968年以降没の作家は保護期間延長により 2038 年まで入らない
- **無料 Web 小説**(カクヨム・なろう等): 現代語。私的使用の範囲でのみ利用
- **購入した現代小説**(紙+OCR/手打ち、または DRM フリー電子書籍): 私的使用の範囲でのみ利用。
  DRM 解除を伴う抽出は私的使用でも違法なので行わない
- ~~WRIME(SNS 感情ラベル付きデータ)~~: 見送り。ラベルが基本 8 感情で、
  本アプリの肝である細かい選び分けには粗すぎるため

### 3.2 データの権利区分(2 パック構成)

| パック | 中身 | 扱い |
|---|---|---|
| 同梱パック | 自作シナリオ+青空文庫由来 | アプリに同梱可。将来の配布・公開も可 |
| プライベートパック | Web 小説・購入本由来 | 端末ローカル読み込み専用。リポジトリ・公開サーバーに置かない |

私的使用(著作権法 30 条)の範囲に収めるため、プライベートパックは
「自分の端末の中だけ」に置く。将来アプリを配布する場合はプライベートパックを外すだけでよい。

### 3.3 抽出パイプライン

1. **収集**: 供給源からテキストを取得(青空文庫は XHTML/テキストをダウンロード)
2. **候補抽出**: 感情語彙リスト(§5.1 で作るもの)で本文を検索し、
   感情語が明記された箇所の前後の文脈(会話+地の文)を切り出す
3. **フィルタ**: 「感情語を隠しても文脈から推測可能か」
   「場面が自己完結しているか」「現代の読者に通じるか」を LLM+人手で選別
4. **整形**: 感情語をマスクして出題文に、元の語+類義語を正解セットにする

```json
{
  "id": "n001",
  "source": { "work": "作品名", "author": "著者名", "aozoraUrl": "..." },
  "passage": "……(感情語を〈？〉にマスクした場面文)……",
  "answer": { "word": "悔しい", "synonyms": ["無念", "口惜しい"] },
  "originalSentence": "(マスク前の原文)",
  "explanation": "どの描写が手がかりだったか"
}
```

留意点:
- 文体が古い作品が多いため、フィルタで現代でも読みやすい場面を優先する
- 一部の青空文庫作品は著作権継続(許諾公開)のため、**著作権切れ作品のみ**使う

### 会話シナリオ(A)の方針(要議論)

- カテゴリ: 職場 / 家族 / 友人 / パートナー あたりを想定
- 難易度:
  - ★1 感情が比較的明白(単一感情)
  - ★2 候補が競る(似た感情語の選び分け)
  - ★3 混合感情・建前と本音のズレ(言葉と感情が一致しない発話)
- 作り方: LLM で下書きを一括生成 → 人手で監修・調整、が現実的
- v1 の目標数: 30〜50 本程度?

## 4. 採点方式(ハイブリッドで決定)

1. **自動照合**: 回答テキストに `best/ok/poor` の `word` または `synonyms` が含まれるかを判定
   - best に一致 → ◎ / ok に一致 → ○ / poor に一致 → △
2. **自己採点フォールバック**: どれにも一致しない場合、答えを見た上で自分で ◎/○/× を選ぶ
   - 用意した語と違っても適切なラベルはあり得るため、自己採点を「逃げ」ではなく正規の手段とする

## 5. 画面構成(最小)

1. **ホーム**: 「練習する(会話)」「小説モード」「復習する」「感情語彙図鑑」「成績」
2. **練習画面**: コアループ(出題→入力→答え合わせ)。会話/小説の 2 モード
3. **復習画面**: 外した問題・自己採点×の問題を再出題(簡易間隔反復)
4. **感情語彙図鑑**: 感情語を系統別に一覧。各語に「どういう時に刺さるか」の短い説明。
   分類の枠組みは中村明『感情表現辞典』の 10 分類
   (喜・怒・哀・怖・恥・好・厭・昂・安・驚)をベースにする(§5.1 参照)
5. **成績**: 連続練習日数、正答率、感情系統別の得意・不得意

### 5.1 感情語彙の出典

既存の感情辞書を参考にして語彙リストを自作する(辞書そのものの転載はしない):

- **中村明『感情表現辞典』(東京堂出版)**: 感情語を 10 分類+複合に整理した定番の辞典。
  約 2,150 キーワード。→ 図鑑の**分類の枠組み**として採用
- **[JIWC-Dictionary](https://sociocom.naist.jp/jiwc-dictionary/)(NAIST)**:
  クラウドソーシングで構築された日本語感情表現辞書(悲しみ・不安・怒り・嫌悪・信頼・驚き・喜びの 7 感情)。
  データとして公開されている。→ 語彙の収集・網羅性チェックに利用
- 図鑑に載せる各語の「どういう時に刺さるか」の説明は、シナリオと同様に LLM 下書き+人手監修で自作する

## 6. 非機能要件

- 実装方式は **PWA**(= 普通の Web アプリをスマホのホーム画面に追加してアプリのように使う方式)。
  App Store 審査・開発者登録が不要で、iPhone でも Android でも動き、
  オフライン動作・ローカル保存という本アプリの要件と相性が良い
- オフラインで練習可能(シナリオ同梱、記録はローカル保存)
- アカウント登録なし、外部送信なし

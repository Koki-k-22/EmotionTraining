
# 感情の「深層」を言語化するための理論的枠組み

調査時点：2026年7月11日

## 0. 結論

「ムカついた」から「プライドが傷ついた」「信頼されていないと感じた」のような具体的心情を生成するには、単一の感情分類表だけでは不十分である。

最も実用的なのは、次の4層を組み合わせたモデルである。

> **表層的な感情表現**  
> → **一次的な感情・傷つき（EFT）**  
> → **状況をどう解釈したか（認知的評価理論）**  
> → **何が大切だった／満たされなかったか（NVC）**

そして、その推測を相手に返す方法を、Motivational Interviewing（MI）のcomplex reflectionとして設計する。

具体的には、

> 「ムカついた」  
> → 怒り  
> → 軽視・不当扱い・裏切り・妨害などの評価  
> → 傷つき／恥／恐れ／無力感  
> → 尊重・信頼・公平・自律性などのニーズ  
> → 「自分のことを軽く扱われたようで、傷ついたんですね」

という構造である。

ただし、**怒りの背後には必ず傷つきや恐れがある、という一対一の法則は存在しない**。EFTでも、怒りは二次感情の場合だけでなく、権利侵害に対する一次的・適応的な怒りの場合がある。したがってアプリの学習目標は「正解を言い当てる」よりも、**文脈に即した複数の仮説を、断定せず共感的に提示する能力**と定義するのが妥当である。

## 1. 「深層心情」という語の整理

研究上、「深層心情」は標準化された単一の専門概念ではない。少なくとも次の異なるものが混在している。

| 層 | 内容 | 「ムカついた」の例 |
|---|---|---|
| 表出 | 発話・態度として表れた感情 | 腹が立った、ムカついた |
| 一次感情 | より直接的な情動反応 | 傷ついた、怖かった、恥ずかしかった |
| 評価・意味 | 出来事をどう解釈したか | 軽く扱われた、裏切られた、不当に扱われた |
| ニーズ・関心 | 何が大切だったか | 尊重、信頼、公平、安全、自律性 |
| 自己・関係の意味 | 自分や関係について生じた意味 | 大切にされていない、能力を認められていない |
| 行動傾向 | 何をしたくなったか | 抗議したい、距離を置きたい、身を守りたい |
| 対人的機能 | 感情表出が相手に及ぼす機能 | 相手を止める、譲歩させる、助けを得る |

アプリではこれらをすべて「隠れた感情」と呼ぶのではなく、少なくとも以下を分離した方がよい。

- `primary_emotion`：一次感情
- `appraisal`：状況の意味づけ
- `need_or_concern`：ニーズ・価値・関心
- `self_relational_meaning`：自己・関係についての意味
- `action_tendency`：行動傾向
- `interpersonal_function`：感情表出の対人的機能

「信頼されていないと感じた」は狭義の感情語ではなく、主として**関係についての認知的評価**である。一方、「傷ついた」「恥ずかしかった」は感情に近く、「信頼してほしかった」はニーズ・願いに近い。この区別はQAの品質管理上、非常に重要である。

## 2. Emotion-Focused Therapy（EFT）

### 2.1 基本的区別

GreenbergのEFTでは、感情反応を主として次のように区別する。

#### 一次感情

状況に対する、最も直接的で基本的な最初の反応である。例えば、喪失に対する悲しみ、危険に対する恐れ、不当な侵害に対する怒りなどである。

さらに一次感情は次の二つに分けられる。

- **一次的・適応的感情**  
  現在の状況について有用な情報を与え、適切な行動を準備する。例：不当扱いに対する自己防衛的な怒り。

- **一次的・不適応的感情**  
  過去の傷つきや情動スキーマによって反復され、現在の状況に十分適合しない。例：「自分は価値がない」という慢性的な恥、「結局見捨てられる」という恐れ。

Greenbergは、一次感情が行動や問題解決に有用な情報を含む一方、反復する中核的な恥や無価値感などは変容の対象になると説明している。[Greenberg, 2004](https://www.eftin.nl/media/1061/emotion_focused_therapy_-_greenberg_2004.pdf)

#### 二次感情

別の感情、思考、または複雑な内的過程に対する反応として生じる感情である。典型例は、

- 傷ついたことに対して怒る
- 怖さや恥を隠すように怒る
- 怒った自分に罪悪感を持つ
- 心配を繰り返して不安になる

などである。

Greenbergは、「傷つきへの怒り」「怒りに対する恐れや罪悪感」を二次感情の例として明示し、二次感情から一次的な生成要因を探索する必要があるとしている。また、「自尊心の喪失に関する恥が激怒の下にある」という例も挙げている。[Greenberg, 2004](https://www.eftin.nl/media/1061/emotion_focused_therapy_-_greenberg_2004.pdf)

#### 道具的感情

感情表出が、相手の反応を引き出すための対人的機能を持つ場合である。例えば、

- 怒ることで相手を従わせる
- 悲しみを示して援助を得る
- 恐れを表して相手を引き止める

などである。

必ずしも意識的な「演技」とは限らず、過去に効果があったため習慣化された表出も含む。QAでは「背後の本当の感情」として扱うより、`何を感じているか`と`その表出が関係の中で何をしているか`を分ける必要がある。

### 2.2 「怒りの背後」の構造

EFTから導ける怒りの主なパターンは次の通りである。

| 表出された怒り | 背後にあり得るもの | 心情フレーズ例 |
|---|---|---|
| 拒絶への怒り | 傷つき、悲しみ、見捨てられる恐れ | 「大切にされていないようで傷ついた」 |
| 批判への怒り | 恥、無価値感、能力への不安 | 「自分を否定されたように感じた」 |
| 裏切りへの怒り | 傷つき、喪失、警戒 | 「信頼していた分、裏切られたようでつらかった」 |
| 威圧への怒り | 恐れ、無力感 | 「怖かったけれど、弱い立場に置かれたくなかった」 |
| 不当扱いへの怒り | 一次的・適応的怒り | 「理不尽に扱われ、黙っていられなかった」 |
| 境界侵害への怒り | 自己防衛、尊厳、自律性 | 「勝手に踏み込まれたように感じた」 |
| 相手を動かす怒り | 道具的機能 | 「強く言わないと聞いてもらえないと思った」 |

重要なのは、右側を機械的に「真の感情」としないことである。EFTには、恐れを変容する**一次的・適応的な怒り**もある。Greenbergの症例でも、虐待されたことへの怒りが自己防衛と自己肯定を回復させ、恐れを変容させる過程が報告されている。[Greenberg, 2004](https://www.eftin.nl/media/1061/emotion_focused_therapy_-_greenberg_2004.pdf)

### 2.3 QA設計への利用

EFTは、次の問題形式に適している。

- 表出された感情は一次・二次・道具的のどれが考えられるか
- 二次感情だとすれば、その直前にどんな感情があり得るか
- その感情は現在の状況に適応的か、過去のパターンの反復か
- その怒りは攻撃ではなく、何を守ろうとしているか

ただし、短い一発話だけから一次・二次を確定することはできない。会話履歴、出来事、関係性、身体反応、時間的順序が必要である。

## 3. 非暴力コミュニケーション（NVC）

### 3.1 感情とニーズ

RosenbergのNVCは、コミュニケーションを概ね次の4要素に分ける。

1. 観察：評価を混ぜず、何が起きたか
2. 感情：そのとき何を感じたか
3. ニーズ：その感情に関係する、大切なこと
4. 要望：具体的に何を依頼するか

NVCでは、感情を「ニーズが満たされている／満たされていない状態を知らせるシグナル」として理解する。CNVCの公式リストも、感情語を「ニーズが満たされているとき」「満たされていないとき」に分けている。[CNVC Feelings and Needs Inventory](https://www.cnvc.org/images/pdf/Feelings-Needs-CNVC.pdf)

一方、これは厳密に実証された「特定のニーズが特定の感情を必ず引き起こす」という一対一の因果法則ではない。NVCは主として、自己理解と共感的対話のための実践モデルとして扱うべきである。

### 3.2 感情・評価・ニーズの混同

NVCでは、次の表現は文法上「感じた」となっていても、純粋な感情語というより、相手についての評価や解釈を含む。

- 無視されたと感じた
- 裏切られたと感じた
- 操作されたと感じた
- 信頼されていないと感じた
- 軽く扱われたと感じた

これらはアプリにとって有用だが、`emotion`ではなく`appraisal`または`relational meaning`として持つ方がよい。そのうえで、

> 「信頼されていないと感じて、悲しかった／悔しかった」

のように、評価と感情を接続する。

### 3.3 ニーズ語彙の整理

CNVCのリストには、つながり、尊重、安全、信頼、自律性、意味、能力、誠実さ、遊び、身体的健康などが含まれる。公式リスト自身も、網羅的・決定的な分類ではなく、自己探索の出発点だと明記している。[CNVC Inventory](https://www.cnvc.org/images/pdf/Feelings-Needs-CNVC.pdf)

アプリ用には、以下のようにまとめられる。

| ニーズ群 | 語彙例 | 深層心情フレーズ |
|---|---|---|
| 尊重・承認 | 尊重、評価、受容、自己尊重 | 「自分の努力を認めてほしかった」 |
| 信頼・誠実さ | 信頼、正直さ、一貫性、約束 | 「信頼関係を大事にしていたのに、裏切られたようだった」 |
| つながり | 共感、親密さ、所属、理解 | 「気持ちを分かってもらえず、ひとりになった感じがした」 |
| 安全・安定 | 安心、予測可能性、保護 | 「何が起きるか分からず、身構えていた」 |
| 公平・相互性 | 公平、平等、配慮、相互性 | 「自分だけ損を押しつけられたように感じた」 |
| 自律性・境界 | 選択、自由、空間、独立 | 「勝手に決められて、自分の意思を奪われた感じがした」 |
| 能力・有効性 | 能力、貢献、達成、成長 | 「役に立てないと思われたようで悔しかった」 |
| 意味・明確さ | 理解、目的、明確さ、希望 | 「理由も分からないまま扱われ、納得できなかった」 |
| 休息・身体 | 休息、睡眠、食事、余裕 | 「余裕がなく、これ以上求められるのがしんどかった」 |

### 3.4 QA設計への利用

NVCは、**語彙と回答文の生成テンプレート**として特に有用である。

- 「本当は、〜を大切にしていた」
- 「〜してもらえると思っていた」
- 「〜へのニーズが満たされなかった」
- 「〜を分かってもらえず、寂しかった」
- 「〜を尊重してほしかった」
- 「〜を自分で選びたかった」

ただし「ニーズが満たされなかったから怒った」という一段階だけでは、具体性が不足する。認知的評価を間に挟むとよい。

> 尊重のニーズが満たされなかった  
> よりも  
> 人前で意見を否定され、能力まで低く見られたように感じ、尊重されていないことが悔しかった

の方が、傾聴練習として有用である。

## 4. 認知的評価理論（appraisal theory）

### 4.1 基本的発想

認知的評価理論では、感情は出来事そのものだけでなく、その出来事が本人の目標、自己、関係、対処可能性にとって何を意味するかによって分化すると考える。

SmithとEllsworthは、感情経験を区別する評価次元として、快・不快、予想努力、確実性、注意、自己／他者の責任・統制、状況統制などを抽出した。[Smith & Ellsworth, 1985](https://emotionresearcher.com/wp-content/uploads/2015/08/SE1985.pdf)

SchererのComponent Process Modelでは、概ね次の評価が重視される。

- 新奇性
- 快・不快
- 目標・ニーズとの関連
- 目標との一致／不一致
- 原因と責任
- 対処可能性・統制可能性
- 規範や自己概念との両立

Lazarusは感情を「人と環境との関係の意味」として捉え、怒りでは自己・自尊心への侮辱や侵害、不当な扱いなどを中心的な関係テーマとした。

### 4.2 同じ怒りを下位分類できるか

**理論的・実用的には可能である。ただし、自然種のような固定的下位分類ではなく、評価プロフィールによる分類である。**

Kuppensらは、怒りに関して、

- 目標妨害
- 他者責任
- 不公平
- 統制
- 敵対的行動傾向

を検討した。他者責任や傲慢な権利意識としての不公平は怒りに比較的特異的だった一方、どの単独要素も怒りに必要十分ではなかった。[Kuppens et al., 2003](https://pubmed.ncbi.nlm.nih.gov/14498795/)

したがって「怒り＝不公平」と固定するのではなく、複数軸を持たせるのが適切である。

| 怒りの評価プロフィール | 中核的意味 | 深層心情の例 |
|---|---|---|
| 目標妨害型 | やろうとしていたことを阻止された | 「努力を台無しにされた」 |
| 不公平型 | 自分だけ不利益を負った | 「理不尽に扱われた」 |
| 他者責任型 | 相手が避けられたのにそうしなかった | 「無責任さに腹が立った」 |
| 自尊心脅威型 | 能力・価値・地位を低く扱われた | 「見下されたようで悔しかった」 |
| 信頼侵害型 | 約束・期待・誠実さが破られた | 「信じていた分、裏切られた」 |
| 境界・自律性侵害型 | 選択や私的領域に介入された | 「自分の意思を無視された」 |
| 関係拒絶型 | 大切にされていないと解釈した | 「自分は必要ないようで傷ついた」 |
| 脅威防衛型 | 害・支配・危険を感じた | 「怖くて、自分を守りたかった」 |
| 無力・低統制型 | 変えられない、逃げられない | 「どうにもできず、やり場がなかった」 |
| 高統制・対抗型 | 相手を止められると思った | 「黙っていると押し切られると思った」 |
| 規範違反型 | 礼儀・倫理・役割に反した | 「そんな扱いは許されないと思った」 |

日本でも、三谷・唐沢が376名に13感情の経験を想起させた研究で、快・不快、自己／他者統制、確実性、努力・注意、新奇性、人／状況統制、動機づけ状態、対処可能性の8因子を抽出し、主要感情の評価プロフィールがおおむね海外理論と整合することを示した。[三谷・唐沢, 2005](https://www.jstage.jst.go.jp/article/jjpsy1926/76/1/76_1_26/_article/-char/ja)

### 4.3 QA設計への利用

評価理論は、今回のアプリにおける**推論エンジンの中心**に最も適している。

発話ごとに以下を付与する。

```text
出来事:
誰が何をしたか:

目標・期待:
何を望んでいたか:

評価:
- 目標妨害:
- 不公平:
- 他者責任:
- 自尊心への脅威:
- 信頼侵害:
- 拒絶・所属への脅威:
- 境界・自律性侵害:
- 危険・安全への脅威:
- 統制可能性:
- 確実性:

一次感情候補:
ニーズ候補:
共感的反映文:
```

## 5. Emotional granularityとaffect labeling

### 5.1 Emotional granularity

Emotional granularity／emotion differentiationは、同じ不快感の中でも、怒り、悔しさ、悲しさ、恥、罪悪感、不安、孤独などを区別して経験・記述する能力を指す。

Barrettらの研究では、感情語を一括して「悪い気分」と用いる人と、状況に応じて異なる感情語を使い分ける人の差として検討されてきた。[Barrett et al., 2001の書誌情報](https://journals.sagepub.com/doi/10.1177/0963721414550708)

Kashdan、Barrett、McKnightのレビューでは、感情を細かく区別できることは、強い苦痛時の過度な飲酒、攻撃、自傷などの不適応的行動の少なさや、拒絶への神経反応、抑うつ・不安の重症度の低さと関連すると整理されている。[Kashdan et al., 2015](https://journals.sagepub.com/doi/10.1177/0963721414550708)

ただし注意点がある。

- 多くは相関研究であり、「語彙を教えれば必ず精神健康が改善する」とまでは言えない。
- 感情粒度の測定法によって結果が変わり得る。
- 近年の青年研究では、感情明瞭性には比較的一貫した関連が見られたが、感情分化の効果は多くが非有意だった例もある。[Hohensee et al., 2025](https://pubmed.ncbi.nlm.nih.gov/39325397/)
- 日常的な反復記録そのものが粒度を高める可能性も報告されている。[Hoemann et al., 2021](https://pmc.ncbi.nlm.nih.gov/articles/PMC8355493/)

したがって、アプリの効果指標では、語彙数だけでなく、

- 文脈に合った感情を区別できるか
- 感情と評価とニーズを混同していないか
- 一つの感情に固定せず複合感情を認識できるか
- 推測の不確実性を表現できるか

を評価すべきである。

### 5.2 Affect labeling

Affect labelingは、感情に言葉を与えること自体が情動反応を調整し得る現象である。

LiebermanらのfMRI研究では、感情的な顔に感情ラベルを付ける条件で、他の符号化条件に比べて扁桃体などの反応が低下し、右腹外側前頭前野の活動が増加した。[Lieberman et al., 2007](https://pubmed.ncbi.nlm.nih.gov/17576282/)

後続の4研究では、ネガティブ画像を見るだけの場合より、感情ラベルを付ける場合に自己報告上の苦痛が低く、再評価や注意転換と近い効果が見られた。ただしポジティブ画像の快感情も弱めたため、「ネガティブ感情だけを改善する」というより、情動反応全般を弱める可能性が示されている。[Lieberman et al., 2011](https://pubmed.ncbi.nlm.nih.gov/21534661/)

TorreとLiebermanは、経験・自律神経・神経・行動の研究を整理し、affect labelingを「本人が調整を意図していなくても働き得る暗黙的な感情調整」と位置づけている。[Torre & Lieberman, 2018](https://journals.sagepub.com/doi/10.1177/1754073917742706)

### 5.3 両者の違い

- **Affect labeling**：今の感情に名前を付ける行為と、その短期的効果
- **Emotional granularity**：類似した感情を状況に応じて細かく区別する能力・傾向

したがって、「ムカつく→怒り」と命名するだけでもaffect labelingになり得るが、「悔しい、軽視された傷つき、信頼を裏切られた悲しさ」と区別できることがgranularityに近い。

## 6. RogersとMotivational Interviewing

### 6.1 Rogersの共感的理解

Rogersの中心概念は、技法としての単純な「オウム返し」ではなく、相手の内的参照枠を、あたかも自分のものであるかのように、ただし「あたかも」の性質を失わず理解しようとする共感的態度である。[Rogers, 1975](https://journals.sagepub.com/doi/abs/10.1177/001100007500500202)

Reflection of feelingは、その理解を言葉にして相手に返す方法の一つである。

- 内容の反映：「会議で意見を取り上げてもらえなかったんですね」
- 感情の反映：「悔しかったんですね」
- 意味の反映：「自分の考えには価値がないと言われたように感じたんですね」

Rogersの考え方からは、正確なラベルを当てることより、**相手自身が経験を確かめ、修正し、より深く探索できる関係**が重要になる。

### 6.2 MIのcomplex reflection

MIでは、単純な反映は発話の反復・言い換え、complex reflectionは、発話に明示されていない意味や感情を仮説として加える反映である。

公的なMI解説では、

1. 背後の意味・感情について仮説を作る
2. 会話全体の文脈を考慮する
3. 最善の推測を反映として返す
4. 相手の反応を待つ
5. 間違っている可能性に開かれている

という手順が示されている。[SAMHSA/NCBI Bookshelf](https://www.ncbi.nlm.nih.gov/books/NBK571068/?report=printable)

この方法は今回のQAに直接対応する。

> 発話：「上司にああ言われて、マジでムカついた」  
> 単純な反映：「上司に腹が立ったんですね」  
> 複雑な感情反映：「人前で能力を否定されたようで、悔しかったんですね」  
> 複雑な意味反映：「努力してきたことまで軽く扱われたように感じたんですね」

ただし実会話では、次のような**仮説性を残す表現**も学習対象にすべきである。

- 「もしかすると、〜という感じでしょうか」
- 「〜されたようで、悔しかったのかもしれませんね」
- 「腹立たしさの中に、少し傷つきもあるように聞こえます」
- 「違っていたら教えてください」

## 7. 日本国内の研究・言語資源

### 7.1 日本語感情語の尺度

寺崎・岸本・古賀は、日本語の感情形容詞648語を収集し、大学生による評定と因子分析から、最終的に80項目、8尺度の多面的感情状態尺度を構成した。

- 抑うつ・不安
- 敵意
- 倦怠
- 活動的快
- 非活動的快
- 親和
- 集中
- 驚愕

日本語として自然な感情語候補の供給源になるが、状態測定尺度であり、「怒りの背後の意味」を直接分類する資源ではない。[寺崎・岸本・古賀, 1992](https://www.jstage.jst.go.jp/article/jjpsy1926/62/6/62_6_350/_article/-char/ja)

### 7.2 日本語感情語辞書

安達らの辞書は、6,894見出し語を、

> 喜・怒・哀・怖・恥・好・厭・昂・安・驚

の10カテゴリに対応するベクトルとして保持している。[安達ほか, 2021](https://www.jstage.jst.go.jp/article/jvs/41/161/41_21/_article/-char/ja)

これは入力発話から表層感情候補を抽出したり、誤答選択肢を作ったりする用途に向く。一方、「軽視された」「信頼を失った」といった関係的意味やニーズは別辞書が必要である。

### 7.3 ML-Ask

PtaszynskiらのML-Askは、日本語テキストについて、

- 一般的な感情表現
- 個別感情語
- 感情タイプ
- 感情価・活性度
- 否定などによる極性変化

を分析するオープンソースシステムである。[Ptaszynski et al., 2017](https://cir.nii.ac.jp/crid/1050564289223216256)

表層感情の自動アノテーションや、問題文候補の探索には有用だが、深層推論の正解辞書としては不足する。

### 7.4 JAppraisal辞書

佐野によるJAppraisal辞書は、8,544語義の評価表現を、肯定・否定だけでなく、愛情・倫理などの**評価基準の種類**によって分類している。大規模コーパスを用いた使用傾向の検討も行われている。[佐野, 2012](https://cir.nii.ac.jp/crid/1390290699744574848)、[言語資源カタログ](https://www.gsk.or.jp/catalog/gsk2011-c/)

心理学の認知的評価理論と、言語学のAppraisal Frameworkは同一理論ではない点に注意が必要だが、発話中の「非道だ」「軽率だ」「大切だ」など、**何を基準に相手を評価しているか**を抽出する資源として使える。

### 7.5 関係文脈を含む感情言語化

西田は、主観的感情体験の発話を、

- 他者への評価的な感情・気持ち
- やりとりの中での自己の感情・気持ち
- やりとりの中での他者の感情・気持ち

に分類している。特に「受容されている」「認められている」「信頼されている」といった関係的表現も、感情言語化の一部として扱っている。[西田, 2015](https://www.jstage.jst.go.jp/article/jjdp/26/4/26_300/_pdf)

今回の「信頼されていないと感じた」に近く、日本語QAでは、狭い感情語だけでなく**やりとりの中の自己・他者・関係の意味**を対象にする根拠になる。

ただし対象者20名の探索的研究であり、一般的分類体系として確立済みとは言えない。

### 7.6 国内の傾聴訓練

臨床看護師向けの傾聴プログラムでは、

- 傾聴
- 共感
- 感情の反映
- 意味の反映
- 3人一組のロールプレイ
- 話し手・聴き手・観察者からのフィードバック

が組み合わされ、「感情と意味の反映的態度」の効果が2週間後にも維持されたと報告されている。ただし、対照群のない前後比較であり、効果の因果推論には限界がある。[臨床看護師のための傾聴プログラム試案の検討](https://www.jstage.jst.go.jp/article/njku/25/1/25_9/_html/-char/ja)

アプリ設計上は、単純な正誤問題だけでなく、

1. 回答を生成する
2. 話し手役が「分かってもらえた度」を評価する
3. 観察者が感情・意味・断定性を評価する

というロールプレイ型評価に根拠を与える。

## 8. 枠組みの比較と推奨

| 枠組み | 深層化の単位 | 強み | 限界 | QAでの役割 |
|---|---|---|---|---|
| EFT | 二次感情→一次感情 | 怒りの下の傷つき・恥・恐れを扱える | 文脈なしに一次・二次を確定できない | 感情層の推論 |
| NVC | 感情→ニーズ | 語彙が分かりやすく、回答文を作りやすい | 一対一の実証的因果表ではない | ニーズ語彙・表現テンプレート |
| 評価理論 | 出来事→意味づけ→感情 | 同じ怒りを不公平、妨害、自尊心脅威などに分けられる | 評価項目が多く、自然な応答に変換する必要 | 中核分類・問題生成 |
| 感情粒度 | 粗い感情→細かな区別 | 学習目標として妥当 | 効果研究は相関的なものも多い | 到達度指標 |
| Affect labeling | 感情→言葉 | 言語化自体の調整効果を支持 | 「深層」の正確さを保証しない | 練習の意義 |
| Rogers | 内的参照枠への共感 | 態度・非断定性を重視 | 固定的分類表を提供しない | 応答品質の原則 |
| MI complex reflection | 明示内容→推測された意味・感情 | QA形式に直接対応 | 誤推測を検証する対話が必要 | 回答生成・採点方法 |
| 日本語感情辞書 | 表現→感情カテゴリ | 語彙収集、自動処理に有用 | ニーズ・関係的意味に弱い | 入力解析・語彙候補 |

### 推奨構成

1. **認知的評価理論を中核タクソノミーにする**
2. **EFTで一次感情候補を付ける**
3. **NVCでニーズ・価値候補を付ける**
4. **MIのcomplex reflectionとして自然文にする**
5. **感情粒度を学習成果として測定する**
6. **日本語辞書・コーパスを語彙候補の供給源にする**

## 9. 「深層心情」語彙・フレーズの類型案

### A. 一次感情型

- 傷ついた
- 悲しかった
- 寂しかった
- 恥ずかしかった
- 悔しかった
- 怖かった
- 不安だった
- 心細かった
- がっかりした
- 裏切られたようでつらかった
- 無力に感じた
- 取り残された感じがした

### B. 評価・意味づけ型

- 〜と感じた
- 〜と思われた気がした
- 〜されたように受け取った
- 〜まで否定されたようだった
- 軽く扱われたように感じた
- 見下されたように感じた
- 信頼されていないと感じた
- 大切にされていないと感じた
- 自分だけ不当に扱われたと感じた
- 努力を無駄にされたように感じた
- 仲間として認められていないと感じた
- 自分の意思を無視されたと感じた

### C. ニーズ・価値型

- 〜を大切にしていた
- 〜してほしかった
- 〜を分かってほしかった
- 〜を認めてほしかった
- 〜を尊重してほしかった
- 〜を信頼してほしかった
- 〜について自分で決めたかった
- 〜が守られると思っていた
- 〜へのニーズが満たされなかった
- 〜を期待していた分、つらかった

### D. 脅威型

- 自尊心を傷つけられた
- 自分の能力を疑われたようだった
- 居場所を失うようで怖かった
- 関係が壊れる気がした
- 見捨てられるようで不安だった
- 自分の立場が脅かされた
- 安全ではないと感じた
- 自由を奪われたようだった
- 境界を越えられたようだった

### E. 行動傾向型

- 自分を守りたかった
- 抗議せずにはいられなかった
- 相手を止めたかった
- 距離を置きたかった
- その場から逃げたかった
- 分かってもらうまで訴えたかった
- これ以上傷つかないよう閉じこもりたかった

### F. 複合感情型

- 腹立たしさと同時に悲しかった
- 悔しいけれど、怖さもあった
- がっかりしたし、裏切られた感じもした
- 恥ずかしさを隠すために強く言いたくなった
- 寂しかったが、寂しいと言うのも悔しかった

## 10. QAデータ設計案

### 10.1 推奨データ構造

```yaml
utterance: "あいつ、みんなの前であんな言い方して。マジでムカついた"
context:
  relationship: "上司と部下"
  event: "会議中に提案を強く否定された"

surface_emotion:
  - anger
  - irritation

appraisals:
  - public_devaluation
  - competence_threat
  - disrespect
  - other_responsibility

primary_emotions:
  - hurt
  - shame
  - frustration

needs:
  - respect
  - recognition
  - fairness

relational_meanings:
  - "能力を低く見られた"
  - "人前で大切に扱われなかった"

acceptable_reflections:
  - "みんなの前で能力を否定されたようで、悔しかったんですね"
  - "提案だけでなく、自分まで軽く扱われたように感じたんですね"

overreaching_answers:
  - "子どもの頃から認められなかった傷が刺激されたんですね"
  - "本当は上司が怖いだけなんですね"

simple_only:
  - "怒っているんですね"
```

### 10.2 採点軸

正答一つではなく、多軸採点が望ましい。

- **文脈適合性**：発話に根拠があるか
- **深さ**：表層語の反復を超えているか
- **層の識別**：感情・評価・ニーズを区別できるか
- **具体性**：「つらい」だけでなく何がつらいか
- **非断定性**：相手の内面を決めつけていないか
- **簡潔性**：一度に多くを推測しすぎていないか
- **共感性**：非難や診断になっていないか
- **修正可能性**：相手が「違う」と言える表現か

### 10.3 問題形式

- 最も適切なcomplex reflectionを選ぶ
- 感情・評価・ニーズを分類する
- 表層的な反映を、意味の反映へ書き換える
- 断定的な応答を、仮説的な応答へ修正する
- 同じ「ムカついた」に対する複数の背景を比較する
- 会話の次の発話を見て、最初の仮説を修正する
- 「深すぎる解釈」を見抜く

特に最後の二つが重要である。熟練した傾聴は、最初から正解を透視する能力ではなく、**仮説を返し、相手の反応によって更新する能力**だからである。

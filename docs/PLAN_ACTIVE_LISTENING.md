# Active Listening 深層心情QA 計画書 v0.1(2026-07-11)

既存の「感情語を当てる」練習から一歩進め、**相手の発話の裏にある、より具体的な心情を言語化して返す**
Active Listening(積極的傾聴)の練習QAを新設する。

例: 相手「マジでムカついた」→「プライドが傷ついたんですね」「信頼されていないと感じたんですね」

裏付け調査は codex(gpt-5.6-sol)による3本のレポート(`docs/research/`)。
本計画書はその統合と、本アプリへの落とし込みである。

- [research1_theory.md](research/research1_theory.md) — 感情の深層言語化の理論(EFT・NVC・認知的評価理論・感情粒度・MI)
- [research2_kokugo.md](research/research2_kokugo.md) — 国語の心情把握問題の作問技法(手がかり設計・誤答肢・要素採点)
- [research3_training.md](research/research3_training.md) — 共感反射の訓練・評価手法(Carkhuff・MITI・HRQ・EPITOME)

## 1. 何を鍛えるか(調査からの結論)

### 1.1 スキルの定義

既存モードが鍛えるのは「感情語の選び分け」(Carkhuff でいうレベル3=表層感情の正確な反射)。
新QAが鍛えるのは **レベル4=加算的応答(additive response)**:
発話に明示されていない感情・意味を、根拠のある仮説として言葉にして返す力。
MI では complex reflection、Ivey では意味の反映(reflection of meaning)に相当する。

### 1.2 「深層心情」の構造(中核モデル)

3レポートが共通して支持するのは、感情を1語でなく**連鎖**として捉えるモデル:

```
出来事 → 認識・評価(どう受け取ったか) → 感情 → ニーズ・願い → 発話・行動
```

「ムカついた」の例:

```
みんなの前で企画を一蹴された
→ 努力ごと軽く扱われた/能力を低く見られた(認識・評価)
→ 怒り(表層) の下に 悔しさ・傷つき・恥(一次感情)
→ 尊重してほしかった/努力を認めてほしかった(ニーズ)
→ 「マジでムカついた」(発話)
```

深層心情の言語化とは、この連鎖の**評価・一次感情・ニーズのどれかを言い当てる**こと。
理論的な裏付けと役割分担:

| 枠組み | 本計画での役割 |
|---|---|
| 認知的評価理論(Lazarus, Scherer, Kuppens) | 深層の**分類軸**。同じ怒りを「自尊心脅威/信頼侵害/不公平/軽視/境界侵害…」に下位分類 |
| EFT(Greenberg)の一次/二次感情 | 「怒りの下の傷つき・恥・恐れ」という**層の構造**。★2問題の骨格 |
| NVC(Rosenberg)の感情とニーズ | 「〜してほしかった」系の**語彙と応答テンプレート** |
| MI complex reflection / Carkhuff | **模範応答の形式**と採点ルーブリックの根拠 |
| 国語の心情問題の作問技法 | **手がかりの埋め込み方**と誤答勾配の設計 |

### 1.3 重要な設計上の警告(調査から)

1. **「怒り=傷つき」の一対一法則は存在しない**。不当な扱いへの怒りは一次感情のこともある(EFT)。
   → 深層は問題ごとに文脈で設計し、機械的な変換表にしない
2. **深い解釈が常に正解ではない**。文脈の根拠がない深読みは過剰解釈であり、傾聴では有害
   (Ickes の empathic accuracy: 正解は「本人の内心」にしかない)。
   → 「深すぎる/根拠のない解釈」を毎問 △ に置き、**深読みしすぎも誤答**だと教える
3. **一発話から深層は確定できない**。
   → 国語式に、本文に独立した手がかりを2個以上、異なるチャネル(発話+行動/状況)で埋め込む
4. **断定は減点対象**。実際の傾聴では仮説として返す(「〜んですね」「〜のかもしれませんね」)。
   → 模範応答に仮説マーカーを含める

## 2. 問題形式の仕様

### 2.1 出題形式

既存 CAR(地の文で読む三人称ミニストーリー)と違い、**会話形式**にする:

- **context**: 状況説明(誰が・どんな関係で・何があったか)。1〜3文
- **utterance**: 相手の発話(一人称)。表層感情が言葉・態度に出ている
- **behavior**(任意): 発話に添える観察可能な様子(「…と言って笑ったが、目は笑っていない」等)
- 設問: 「**相手の気持ちを言葉にして返すなら?**」(自由記述、実際に返す一言)

### 2.2 難易度設計

| ★ | 設計 | 例 |
|---|---|---|
| ★1 | 深層が1つで手がかりが明示的。表層感情語は発話にない | 「もう誰にも頼まれてないんだよね、最近」→ 必要とされていない寂しさ |
| ★2 | **表層感情語が発話に明示される**(ムカつく・最悪・もうやだ等)。◎はその下の深層。表層の言い換えは○止まり | 「マジでムカついた」→ ◎軽く扱われた傷つき / ○腹立たしさ |
| ★3 | 発話と感情がズレる(建前・強がり・自嘲)+複合感情 | 「別にいいんだけどね」→ 本当は分かってほしかった |

★2 が本形式の本体。「感情語をオウム返しすると○止まり、裏を言い当てると◎」という勾配が、
既存モードとの違いをそのまま体験させる。

## 3. 採点設計

### 3.1 自動判定(grading.js 拡張)

深層心情は「悔しい」のような1語でなく句なので、**キーワード群マッチ**にする:

- `deep.best[].keys` は OR グループの AND(例: [努力系の語] かつ [軽視系の語] を含めば成立)
- 判定順: deep.best 成立 → **◎** / surface(word+synonyms)一致 → **○**(表層止まり) /
  deep.poor のフレーズ照合 → **△** / どれでもない → 自己採点へ
- 既存の NFKC 正規化を流用。活用語尾を拾うため keys は語幹で書く(「傷つ」「踏みにじ」)

### 3.2 自己採点の強化(調査③の知見)

キーワードマッチは検出信号にすぎない(EPITOME 研究でも ML でさえ誤る)。
不一致時の自己採点は、既存の◎○×ボタンに**チェック観点**を添える:

- 相手の使った感情語の言い換えではなく、その裏の気持ちを言葉にしたか(深さ)
- その推測は発話・状況に根拠があるか(的確さ)
- 決めつけ・助言・説教・「気にしすぎ」になっていないか(伝え方)

3観点は Carkhuff / MITI / HRQ に共通する「深さ・的確さ・伝え方」の分離に対応する。

### 3.3 答え合わせ画面

1. 判定(◎深層に届いた / ○表層止まり / △ズレ)
2. **連鎖の開示**: 出来事→受け取り方→表層の下の感情→ニーズ を図式で見せる
3. cues(発話中の手がかり)と、deep.poor の「なぜ深読みしすぎか」
4. modelReply(仮説マーカー付きの返し方の例)

## 4. 作問要件(docs/AUTHORING_AL.md として別途詳細化)

執筆: Fable / ブラインドテスト・監修: Sonnet / 最終検品: Fable+ユーザー(既存フローを踏襲)

### 4.1 執筆ルール(国語の作問技法の転用)

1. **正解を先に決める**: 内部設計シート(出来事/認識・評価/一次感情/ニーズ/模範反射)を書いてから本文を書く
2. **手がかり2個以上・異チャネル**: 発話の中の理由句(「〜なのに」「〜までして」)+行動・状況描写。
   単一の感情語だけで解ける問題にしない
3. ★2 では表層感情語を**発話に明示**する(隠すのは既存CARの流儀。ここでは逆)
4. **◎は2〜3個用意**する(深層の正解は一意でない。評価型/ニーズ型など層の違う正解を認める)
5. △には**過剰解釈**(本文に根拠のない深読み)と**対象・因果のズレ**を必ず含め、why にパターン名を書く
   (誤答勾配のパターン集は research2 §3.2 を使う: 程度の逸脱/対象のずれ/表面固着/過剰な裏読み/一般論への逃避 等)
6. **時点の固定(矛盾防止)**: 正解は常に**発話時点(いま)の心情**とする。発話が時間経過を語る場合
   (「不安だったけど、見つかってほっとした」)、◎は発話時点に残っている・支配的な心情(安堵、
   あるいは「まだ胸がざわつく」等の余韻)であり、**通過済みの中間感情を◎にしない**。
   中間感情に触れる回答(「不安でしたね」)は、文脈上自然な共感なら○、時点を取り違えていれば△とし、
   why に「時点のずれ」と明記する。執筆時は設計シートに「発話時点の心情」を必ず1行で書き、
   ブラインドテストでもソルバーに「発話時点の心情か」を確認させる
7. 深層心情の語彙・フレーズは新設の `data/deep_lexicon.json`(§5)から選ぶ
8. modelReply は仮説マーカー(「〜んですね」「〜のかもしれませんね」)+断定・助言・評価の禁止。
   文型テンプレは research3 §10 を使う
9. カテゴリ(職場/家族/友人/パートナー)・話者の性別・年代・立場を分散させる

### 4.2 品質ゲート: ブラインドテスト v2

既存プロトコルを深層向けに改訂:

- ソルバー(Sonnet×3、視点別ペルソナ)に **context+utterance+深層心情リスト**(deep_lexicon 由来、40〜60項目)を渡す
- 「この人の**発話時点**の心情の核に最も近いものを順位付きで2つ+**本文中の根拠**」を答えさせる
- 合格: 2/3 のソルバーの top-2 に ◎ 相当が含まれる
- ソルバーが通過済みの中間感情(「不安だったけど→安堵した」の「不安」等)を選んだ場合は
  時点設計の欠陥を疑い、本文か正解セットを修正する
- 根拠が本文で示せない◎(=作問者の思い込み)はこの過程で炙り出される
- FAIL → 手がかりを増強して再テスト、または破棄

### 4.3 v1 の規模(案)

40問: ★1×10 / ★2×20 / ★3×10。まず10問で試作→ブラインドテストの歩留まりを確認してから量産。

## 5. 新しい語彙資産: deep_lexicon.json

執筆・ブラインドテスト・図鑑拡張の共通基盤として、深層心情のタクソノミーを作る。
枠組みは research1 §9 の類型+§4.2 の評価プロフィール+§3.3 のニーズ群:

| 類型 | 例 |
|---|---|
| 一次感情型 | 傷ついた/恥ずかしかった/心細かった/がっかりした/無力に感じた |
| 評価・意味づけ型 | 軽く扱われた/見下された/信頼されていない/大切にされていない/不当に扱われた/意思を無視された |
| ニーズ・願い型 | 認めてほしかった/分かってほしかった/尊重してほしかった/自分で決めたかった/頼ってほしかった |
| 脅威型 | 自尊心を傷つけられた/居場所を失いそうで怖かった/見捨てられそうで不安だった |
| 複合型 | 腹立たしさと同時に悲しかった/恥ずかしさを隠すために強く言いたくなった |

各項目に: phrase / 語幹キー / 類型タグ / どんな発話の裏に潜みやすいか(執筆ヒント)。
既存 emotion_lexicon.json(表層60語)は surface 判定に流用する。
